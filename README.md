# Сервис для создания вопросов для викторин (Quiz API)

## Описание проекта

Данный проект — это REST API сервис для создания и управления викторинами (квизами). Сервис позволяет:

* создавать и управлять категориями вопросов;
* создавать квизы (наборы вопросов);
* добавлять, редактировать и удалять вопросы;
* получать случайные вопросы из квиза;
* проверять правильность ответа пользователя.

Проект реализован на **Django** и следует принципам разделения ответственности:

* слой доступа к данным (services / DAO),
* слой контроллеров (REST API),
* слой сериализации данных,
* слой тестов.

Сервис может использоваться как backend-часть для веб- или мобильного приложения с викторинами.

## Установка и запуск проекта

### 1. Клонирование репозитория

```bash
git clone https://github.com/DamirKFU/higher-web-practice-quiz-python.git
cd quiz-service
```

### 2. Установка зависимостей

Проект использует `pyproject.toml` и пакетный менеджер **uv**:

```bash
uv sync
```

### 3. Применение миграций

```bash
uv run python manage.py migrate
```

### 4. Запуск сервера разработки

```bash
uv run python manage.py runserver
```

После запуска API будет доступно по адресу:

```url
http://127.0.0.1:8000/
```

## Примеры запросов и ответов

### Создание категории

**POST** `/api/category`

```json
{
  "title": "Python"
}
```

**Ответ:**

```json
{
  "id": 1,
  "title": "Python"
}
```

---

### Создание квиза

**POST** `/api/quiz`

```json
{
  "title": "Основы Python",
  "description": "Викторина для начинающих"
}
```

**Ответ:**

```json
{
  "id": 1,
  "title": "Основы Python",
  "description": "Викторина для начинающих"
}
```

---

### Создание вопроса

**POST** `/api/question`

```json
{
  "quiz_id": 1,
  "category_id": 1,
  "text": "Что выведет print(2 + 2)?",
  "options": ["3", "4", "5"],
  "correct_answer": "4",
  "difficulty": "easy"
}
```

**Ответ:**

```json
{
  "id": 1,
  "text": "Что выведет print(2 + 2)?",
  "options": ["3", "4", "5"],
  "correct_answer": "4",
  "difficulty": "easy"
}
```

---

### Проверка ответа

**POST** `/api/question/1/check`

```json
{
  "answer": "4"
}
```

**Ответ:**

```json
{
  "result": true
}
```

## Архитектура проекта

* **models** — описание моделей базы данных
* **services** — слой работы с БД (реализация интерфейсов Abstract*Service)
* **serializers** — преобразование моделей в JSON и обратно
* **views / controllers** — REST API контроллеры
* **urls** — маршрутизация
* **tests** — модульные и интеграционные тесты

Контроллеры не обращаются к базе данных напрямую — вся бизнес-логика вынесена в сервисный слой.
